Средства вычислительной техники (СВТ):
-ЭВМ
-Вычислительные комплексы (ВК)
-Вычислительные системы (ВС)
Всякие считалки, которые обеспечивают обработку данных
-----------------------------------------------------
Средства телекоммуникаций (СТК):
-совокупность каналов связи (провода)
 и каналообразующий аппаратуры (модемы. Они и вызывают передачу
информации из одной точки в другую)
также свичи, шлюзы, файрволы
Реализует передачу данных
------------------------------
Сеть ЭВМ = СВТ + СТК
-------------------------------------
Для того чтобы организовать веб-сервер:
-веб-сервер, обрабатывающий запросы http
-прикладной сервер, на котором содержиться приложение
-база данных (обрабатывает запросы данных)
----------------------------------------------
Разница между LAN (локальная сеть) и WAN (глобальная)
заключяется в используемых протоколах и можемах
протоколы LAN:
-ethernet
-wi-fi (семейство)
протоколы WAN:
-информация передается из точки в точку
Основная разница: 
в LAN передача данных мгновенная, а в WAN за счет конечности
скорости света имеется задержка между передачей информации и
её приемом (времена прохождения пакетов имеют значение)
То есть мы накидали в канал пакетов, а ответы на эти пакеты
приходят пачками (чтобы максимально загрузить канал связи)
-----------------------------------------
PAN - personal area network - всякие блютузы, наушнички, то,
что крутится вокруг нас (те же сетевые протоколы, кстати)
LAN - local (вот стоит у меня в квартире роутер, 3 пк, смартфон)
MAN - metropolian (сети города, где применяются технологии
LAN  и WAN)
WAN - world глобальные
----------------------------
Стандардый объем пакета - 1500 байт
--------------------
в каждом пакете есть свой заголовок и свой концевик
концевик (контрольная сумма, хэш) - сумма байт (все сообщение
разбивается на байты и они скд=ладываются) пакета
 (с учетом переноса)
информация бьётся на пакеты, чтобы обеспечить равный доступ
----------------------
протокол tsp занимается разбиением сообщения на пакеты
и добалением заголовка и концевика
--------------------
точка взаимодействия состоит из ip адреса и порта
порт - уникальное число, которое идиентифицирует программу
на стороне компьютера
Пример:
браузер переходит по какой то ссылке,
-зашифровывает в пакет ip адрес сайта
-порт, на который вы идете (у гелиоса 2222, у веб-серверов 80
443) 
-------------------------------------
гнездо: ip + порт уникально идеентифицирует 2 программы
на 2-х машинах
Соединение на уровне TCP - это соединение между 2 гнездами
------------------------------
                       Транспортный уровень (TCP)
на уровне TCP сообщение бьется на пакеты данных
также организует виртуальный канал (отправляется запрос на 
открытие, возвращается ответ, начинается обмен данными, 
потом соединение закрывается)
----------------------------------
                 Сетевой уровень (IP)
передает пакеты между узлами сети
На данном уровне все пакетыы данных и служебные пакеты
преобразуются в IP data groundes (+служеб инф + IP)
--------------------------------------------
          Канальный уровень (эзернет)
обарачивает ещё раз
отдает сигнал в передающую среду (кабель, провод) 
передает с 1-ой физической машины на другую
------------------------------------------------
физический адрес - 6 байт информации (выдается на производстве)
Внутри одного сегмента локальной сети все адреса уникальны
каждому ip сопоставлен мат адрес
то есть есть утилитка arr, ну или можем отправить в интернет
вопрос - "кто с таким то ip адресом?" в ответ придет
ethernet адрес
--------------------------------------------
IP - 32 разрядное число
первая половинка - номер сети для групповой маршрутизации
вторая половинка - номер хоста

к каждого компьютера есть таблица маршрутизации
